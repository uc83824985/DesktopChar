# Renderer process

负责画布、桌面交互和模块装配。UI 通过领域接口控制 Avatar，不直接访问 Live2D SDK 私有成员。

桌面模式将全局指针换算为窗口局部坐标，并持续查询最终 Pixi 帧上该位置附近的像素 alpha。查询按渲染帧流水线提交多笔 PBO/fence 读回，不等待上一笔完成；结果严格按提交帧序消费，因此鼠标不动时也能捕捉待机动画移入或移出。坐标改变时只允许最新查询更新状态。为避免高 DPI 下单个物理子像素与肉眼所见的 CSS 像素边界不一致，前台读取 `3×3` 设备像素并取最大 alpha，以 `8/255` 过滤几乎不可见的纹理/抗锯齿余量。首个可见样本立即进入，连续 3 个透明样本后退出，pending 不会清空上一次确认结果。只有确认可见的像素会请求 main 恢复窗口交互；透明像素保持穿透。模型 `HitAreas` 只用于给点击附加语义标签，不再决定是否命中。

点击只发送 Runtime Event，拖动只请求 main 更新窗口 bounds。拖动开始后由 pointer capture 和 main 的 drag state 锁定交互，迟到的像素读取不能中断拖动。
